# WT-626 — управление звуком просмотра

## Цель

Сделать звук фильма управляемым и восстанавливаемым для обоих участников
совместного просмотра. Пользователь должен видеть текущую громкость, иметь
mute/unmute и не оставаться без понятного действия, если браузер заблокировал
autoplay аудио у гостя.

## Реализовано

- В stage добавлены локальные mute/unmute и регулировка громкости от `0` до
  `100%` для host preview и guest playback.
- Контрол активируется только когда фактически опубликована или получена audio
  дорожка; для видео без звука он честно отключён с понятным пояснением.
- Действие «Включить звук» у guest повторно вызывает `audio.play()` в
  пользовательском жесте. Это позволяет восстановить звук после autoplay-block
  Chrome/Edge без перезагрузки или переподключения к комнате.
- Громкость и mute являются локальными настройками участника: они не меняют
  воспроизведение, звук и настройки другого человека.

## Проверки

```bash
pnpm --filter @watch-together/frontend test
pnpm check
pnpm test:e2e
```

Unit-тесты покрывают публикацию двух дорожек, guest audio replay после
autoplay-block и изменение громкости у host/guest stage.

## Ограничения

- Физическую доставку и декодирование аудио между Windows и macOS нужно
  подтвердить отдельным LAN-прогоном с реальным файлом.
- Если в выбранном файле нет аудиодорожки либо браузер не отдаёт её через
  `captureStream()`, control останется отключённым: громкость не может создать
  отсутствующий аудиотрек.
